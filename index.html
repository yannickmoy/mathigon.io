Array Functions

Core.js and Fermat.js contain a variety of functions which can be used to simplify the creation and modification of arrays.

tabulate(Math.random, 5);
  // [0.7068, 0.9314, 0.6050, 0.9042, 0.2278]

flatten(tabulate((x, y) => '' + x + y, 3, 3));
  // ['00', '01', '02', '10', '11', '12', '20', '21', '22']

map((x, y, z) => x + y + z, [1, 2, 3], [4, 5, 6], [7, 8, 9]);
  // [12, 15, 18]

list(9).chunk(3);
  // [[0, 1, 2], [3, 4, 5], [6, 7, 8]]

Fermat.js also provides an Array subclass, Vector which is more powerful but also less efficient than native JavaScript arrays.

Expression Parsing

Fermat.js is able to parse complex mathematical expressions, including arbitrary functions, arrays and strings. When evaluating, expressions have access to all constants and functions in Math and M, and you can pass in an object of custom variable and function values.

let expr = Expression.parse('1 * (2 + x) - 4! + norm([1, 2, x^2]) - (100 / 2)%');
expr.evaluate({ x: 10 });

DOM Selection

Boost.js provides a variety of functions to select DOM elements. $, $I, $C and $T return a single element, while $$, $$C and $$T return arrays of elements. $N and $$N can be used to create new DOM elements.

var $el = $C('my-class');
$el.on('click', function() { ... });
$el.transform('scale(2)');

Speical Events

Boost.js seamlessly supports a number of special events. The click event is overwritten to happen on mouseup or touchend to avoid delays on mobile devices. The scroll, scrollStart and scrollEnd events are fired for mouse and touch scroling using requestAnimationFrame. Mathigon also simulates cross-platform pointerStart, pointerMove, pointerEnd, pointerEnter, pointerLeave and pointerOver events.

$element.on('click', function(e) { console.log('click'); });
$element.on('scroll', function(e) { console.log(e.top); });
$element.on('pointerStart', function(e) { console.log(x.top); });

Animations and Effects

Boost.js contains a variety of element transitions, including .enter(), .exit(), and effects like .pulsate(). You can also define generic element animations which use CSS transition to animate any css properties. In particular, you can animate to and from height: auto and width: auto.

$element.fadeIn(200);
$element.pulsate();
$element.exit('fade', 300, 100);  // duration 300ms, delay 100ms

$element.animate([
  { css: 'background', from: '#C00', to: '#0C0', duration: 500, timing: 'linear', delay: 200 },
  { css: 'width', to: '200px' }
]);

M.animate is a wrapper for requestAnimationFrame that can be used to call a function at every step during an animation of fixed length, or indefinitely.

animate(function(prop, time) {
  $element.css(left, prop * 200 + 'px');
  $element.text(time + 's');
}, 500);

UI Elements

Slate.js creates a number of different UI and media element classes. All classes contain appropriate methods and they fire a variety of useful events.

var mySlider = new M.Slider($C('track'), 10);
mySlider.on('move', function(n) { ... })

var myGallery = new M.Gallery($C('gallery'), { callback: function(index) { ... } });
myGallery.next();

M.audio.load('/my-audio.mp4', 'file1');
var myAudio = new M.audio.Chunk('file1', [20, 40]);
myAudio.on('timeupdate', function() { ... });
myAudio.play();

Roadmap

  • Update special events (click, scroll and pointerMove) to allow removing of events.
  • Improve expression parsing, simplification and evaluation
  • Improve geometry functions, allowing arbitrary intersection, projections of geometric objects.
  • Create Bignum library that can handle arbitrarily large integers, arbitrarily accurate decimals and exact fractions. Update all mathematical functions to seamlessly handle Bignum and native Number inputs.
  • Create new SVG library that integrates with the existing geometry, animation and events functions.
  • Create a variety of new reusable UI components. Convert all UI widgets into HTML tags utilising Web Components and Shadow DOM.

Contributing

Mathigon JS was created to simplify web development, to share code between Mathigon websites, and to reduce the need for a large number of external JavaScript libraries and frameworks. While most of the code was written for specific Mathigon uses, rather than to build a generic and universal library, we hope that you will find it useful for animation and computation heavy projects.

To contribute code, please fork our GitHub repository and send us a pull request. If you find bugs or mistakes in Mathigon JS, please email us with details on how to reproduce them.
